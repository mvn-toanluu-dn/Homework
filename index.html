What are the differences between a variable that is: `null`, `undefined`?

Null có nghĩa là giá trị rỗng hoặc giá trị không tồn tại, nó có thể được sử dụng để gán cho một biến như là một đại diện không có giá trị.
Undefined có nghĩa là không xác định, khi bạn khai báo một biến nhưng chưa gán giá trị cho nó, giá trị của biến đó sẽ là undefined.
typeof undefined // undefined
typeof null // object

What is `use strict`? what are the advantages and disadvantages to using it?

Nếu bạn đặt "use strict" ở đầu đoạn code (hoặc hàm), thì JS sẽ mặc định nó là chế độ nghiêm ngặt.
Chế độ này gây ra nhiều lỗi hơn và vô hiệu hoá một số tính năng nhằm nỗ lực làm cho code của bạn trở nên
mạnh hơn, dễ đọc hơn, và chính xác hơn.
Lợi thế:
Nó ngăn chặn hoặc báo lỗi khi các hành động "không an toàn" được thực hiện.
Nó vô hiệu hoá các tính năng khó hiểu hoặc thiếu suy nghĩ.
Nó bắt lỗi vào những tình huống khó xử, không thực thi.
Nhược điểm:
Khi kết hợp code giữa use strict và normal mode, nếu bạn sử dụng thư viện của strict mode, nhưng đã quen với normal mode sẽ có một số hành động trên thư viện không hoạt động như mong đợi.
Tệ hơn nữa là khi đang ở normal mode, bạn sẽ không có lợi thế về các lỗi bổ sung được báo mà nó xảy ra âm thầm.

'4' == 4 // return true
'===': Dùng so sánh giá trị, kiểu dữ liệu.
'4' === 4 // return false
